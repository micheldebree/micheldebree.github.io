Retropixels 0.7.1 released

2020-05-21T19:07:49+02:00

Today I release retropixels version 0.7.1.

Retropixels is a cross-platform command line tool to convert images to a format that is supported by the legendary Commodore 64 home computer.

For this release I have been playing around with different palettes and colour space conversions.

In this new version, you can choose between a couple of colour palettes, and also a colour space to use when the image is quantized, resulting in two new arguments:

--palette <colorodore|pepto|deekay>
--colorspace <xyz|yuv|rgb|rainbow>

Determining the colour of a pixel

Retropixel determines the colour of each pixel in the resulting image by quantization, meaning that the big range of colours in the input image needs to be mapped to the fixed 16 colour palette in the Commodore 64. This is done by calculating the Euclidian distance between the input colour, and each colour in the palette, and picking the colour from the palette that has the lowest distance, so is closest to the original input colour.

So how do you measure the distance between colours? Retropixels uses the Jimp to read the input images, which results in a big array of pixels, where the colour of each pixels is represented by 3 numbers: R (red), G (green) and B (blue). These are all numbers between 0 and 255, which we can use in the Euclidian distance formula. This is really just the Pythagorean theorem that you probably know or have heard of.

Representing a colour with red, green and blue components is called the RGB colour model. And this is where it gets complication, to me at least. There are multiple colour models, and multiple ways of interpreting these models, called colour spaces. I have found, by playing around, that converting from the RGB colour space to another colour space before quantizing, gives different results.

Palette

Prior to 0.7.1, Retropixels used the pepto palette by Philip “Pepto” Timmermann. When you read the article , there seems to be a lot of science behind it, and I liked how it looked. Later I found out that Philip revisited that science and came up with the improved colodore palette. This is now the new default palette.

An argument to the command line utility has been added to select the palette to use. I also added a palette by Deekay.
